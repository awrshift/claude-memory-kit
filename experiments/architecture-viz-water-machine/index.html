Memory Kit · архитектура v0.1 (метафорический прототип)

Как агент помнит
и думает.

У вас есть ИИ-ассистент. Вы хотите дать ему весь контекст проекта — бренд, клиентов, прошлые кампании, правила. Он тонет. Тот же ассистент на следующий день всё забывает. Эта страница объясняет, почему так — и что делает Memory Kit, чтобы собрать из этого работающую систему памяти.

Именно это происходит, когда вы скидываете всю работу в один чат. Старое вытесняется новым — агент забывает первую страницу брифа, путает факты, ломает тон. Одна общая память не масштабируется.

Ответ — не делать колбу больше, а построить машину из разных сосудов. Каждая информация живёт на своём уровне и поднимается наверх только когда нужна.

2Машина памяти

Представьте себе прибор.

Стеклянная колба сверху, под ней — ряды сосудов разных форм, внизу массивный резервуар, ещё ниже — осадок. По трубам течёт вода. Есть манометр, шестерни, насос. Каждый сосуд хранит свою воду, свой тип информации. Всё вместе — то, что агент помнит и чем думает.

⟡ Переход

NSP — записка от вчерашнего себя

Один абзац перед машиной: где вчера остановились, что делать сегодня. Агент читает её первой — не нужно каждый раз вводить в курс.

⟡ План на сегодня

Backlog — список задач

Живой чек-лист текущего проекта. Галочки, новые пункты, приоритеты. Агент видит его сразу — знает, над чем вы сейчас работаете.

L1 Hot Path

Рабочая память

Верхняя колба. Всегда с агентом. Два файла внутри — агент читает их первыми на каждой сессии:

CLAUDE.md ДНК проекта. Кто вы как команда, как общаетесь, какой тон, что нельзя делать. Стабильный — меняется редко, когда бренд сам меняется.

MEMORY.md Короткая память. Свежие паттерны и пометки «вот так мы делаем» из последних сессий. Обновляется часто — после каждого ощутимого вывода.

L2a Rules

Короткие правила

Жёсткие запреты. «Не пиши так», «не используй это». Поднимаются только когда агент коснулся темы.

L2b Ролевые справочники

Ролевые справочники

Как думает дизайнер, редактор, маркетолог. Не запреты, а советы — как сделать хорошо. Каждая роль в своём сосуде. Агент сам подключает нужный справочник, когда разговор попадает в его тему.

L3 Deep Memory

Глубокая память

Большой резервуар. Позиционирование, история решений, канонические правила — то, что проверено временем.

L4 Sediment

Архив

Осадок. Прошлые сессии, устаревшие варианты. Автоматически не поднимается — только если явно спросить.

Маршрут запроса

Выберите запрос маркетолога — агент слышит ключевое слово, и в машине «отзываются» только нужные сосуды. Остальные спокойно стоят. В этом идея экономии контекста.

3Как управлять машиной

Семь рычагов.

Память — это здание. Но здание работает только если есть, кому его использовать. В Memory Kit роль «операторов» играют шесть slash-команд (их набираете вы сами, когда нужно) и один хук (срабатывает автоматически при старте сессии). Всего семь, каждый делает ровно одну вещь. Агент ничего не запускает по своему расписанию.

Что такое сессия для агента Сессия — это одно контекстное окно, обычно 300-500 тысяч токенов из миллиона. Когда окно заполняется, агент сам просит начать свежую сессию (сохранит всё важное в NSP + MEMORY перед этим). За день сессий может быть 3-5-10 — зависит от интенсивности работы. `/close-day` закрывает ВСЕ сессии дня, не одну.

/close-day

Закрыть день

Когда закончили работать на сегодня — набираете /close-day. Агент собирает все сессии дня в дневник и проводит аудит: сравнивает свежие паттерны с короткой памятью и ролевыми справочниками (reference skills), предлагает промоушен голосом. Вы говорите «да/нет», агент пишет сам. Файлы вручную не открываете никогда.

читаетсессии дня + короткую память + справочники
пишет вдневник + память + справочники (по согласию)

/memory-compile

Собрать гайды

Когда накопились дневники и хочется упорядочить (обычно раз в неделю-месяц) — набираете /memory-compile. Агент пересобирает дневники в структурированные тематические гайды.

читаетdaily/*
пишет вknowledge/concepts/

/memory-query

Спросить у памяти

Набираете /memory-query и задаёте вопрос естественным языком. «Что мы решили про SEO для агентств?» — агент отвечает со ссылками на источники.

читаетвсю память
возвр.ответ + ссылки

/memory-lint

Проверить здоровье

Когда кажется, что база «поплыла» — набираете /memory-lint. Семь проверок здоровья: битые ссылки, заброшенные файлы, статьи без заголовка, потерянные перекрёстные ссылки. Отчёт, и всё — ничего не меняется.

читаетструктуру
возвр.отчёт из 7 тестов

/memory-audit

Найти переросшие справочники

Набираете /memory-audit, когда хотите проверить, не пора ли разделить какой-то справочник. Агент сканирует .claude/skills/*-guidance/SKILL.md — если вырос выше ~500 строк и внутри читаются 2-4 независимые темы, предлагает разделить (Scene 6 «Мейоз»).

читает.claude/skills/*-guidance/SKILL.md
возвр.список файлов к разделу

/tour

Экскурсия по системе

Набираете /tour — и агент водит вас по вашим же файлам, объясняет, где что. Когда новый человек заходит в проект, это его первая команда.

читаетвсю систему

Утренний запуск

Не команда, а автоматический механизм. При старте каждой сессии подтягивает NSP, горячую память и свежие дневники в рабочую камеру. Вы ничего не делаете.

Общее · shared

Память агента всегда одна

ДНК агента, правила, операторы, канонические концепты — всё это общее для всех проектов. Агент приносит одну и ту же профессиональную хватку на каждый из трёх клиентов.

Проектное · project-scoped

У каждого проекта свой BACKLOG.md

Ключевой файл проекта — projects/<клиент>/BACKLOG.md с планом задач. Рядом лежат клиентские брифы, бренд-гайды в PDF, архивы кампаний, обсуждения. Агент подтягивает только нужную папку, когда вы говорите «работаем над клиентом А».

5Жизненный путь капли

Как случайное наблюдение
становится правилом.

Свежее наблюдение (жидкость) — это разовая правка в брифе. Повторённый урок (густой янтарь) — частый фидбек, который арт-директор проговаривает на каждом ревью. Проверенное правило (кристалл) — утверждённый закон бренда. На кристалл агент опирается железобетонно.

Фаза 1
Жидкость
daily/*.md

Наблюдение из сессии. Течёт легко, может испариться.

Фаза 2
Янтарь
MEMORY.md

Паттерн замечен повторно, записан в короткую память. Ещё не канон.

Фаза 3
Справочник
.claude/skills/design-guidance/SKILL.md

Канон роли. Агент записал после вашего «да» на `/close-day`-аудите. Поднимается, когда агент работает в этой роли (auto-load via description).

Фаза 4
Кристалл
.claude/rules/*.md

Закон бренда. В жёстких правилах проекта — автозагружается агентом всегда. Меняется редко, когда сам бренд меняется.

Как наблюдение поднимается по лестнице

Упомянуто один раз — остаётся в дневнике сессии. Кандидат, не более.

Повторилось в той же сессии — агент записывает в короткую память (MEMORY.md) как помеченную датой строку.

На `/close-day` агент замечает повтор через сессии или дни — сравнивает свежее с MEMORY.md и ролевыми справочниками (reference skills), сам предлагает голосом: «вижу это в третий раз за неделю — добавить в design-guidance?» Вы говорите «да», агент пишет patch в файл сам. Файлы вы не открываете — только говорите.

Полгода стабильно без противоречий — на следующем `/close-day` агент предлагает перевести паттерн в жёсткие правила (.claude/rules/). После подтверждения — это закон бренда, на который агент ссылается железобетонно.

6База знаний сама наводит порядок

Когда один справочник
разрастается до трёх.

Сосуды со временем наполняются. Ваш ролевой справочник дизайнера (`.claude/skills/design-guidance/SKILL.md`), например, переполняется десятками новых правил и исключений — и внутри появляются три самостоятельные темы. Агент замечает это первым, сам предлагает разделение по смыслу. Вам остаётся только подтвердить голосом. Это не админская задача, а «привести шкаф в порядок» одним словом «да».

1 · Когда замечает

Во время /memory-audit (или /close-day-аудита) агент проверяет размер каждого ролевого справочника. Если файл вырос больше ~500 строк и внутри читаются 2-4 несвязанные темы — это сигнал к разделу.

2 · Как определяет темы

Агент читает справочник и группирует секции по смыслу. Секции, которые почти не ссылаются друг на друга — это независимые темы. Обычно их 2-4 (не всегда три!).

3 · Что делает после «да»

Создаёт N новых справочников: .claude/skills/design-type-guidance/, design-color-guidance/, design-motion-guidance/ — и переносит соответствующие секции по темам. Исходный .claude/skills/design-guidance/ остаётся как оглавление со ссылками на новые.

Ролевой справочник .claude/skills/design-guidance/SKILL.md · 847 строк / ~30 страниц · агент предлагает разделить

Агент заметил:

«В этом гайде три самостоятельные темы — типографика, цвет, анимация. Они почти не ссылаются друг на друга. Разделить по темам? Каждая станет подниматься только когда о ней спросят.»

Готово.

Справочник аккуратно разложен на три раздела по смыслу. Агент теперь грузит только нужный кусок — меньше шума, точнее ответы. Исходный файл остался как оглавление со ссылками: ни одна мысль не потеряна.

Хотите собрать такую память для своего агента?

Memory Kit — это открытый набор файлов и скриптов, который превращает Claude Code в агента с постоянной памятью. Никакой базы данных, только Markdown и несколько Python-хуков. Работает на существующей подписке, без API-расходов.
